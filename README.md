# Hi there 👋 I'm Bigduu

**AI Agent developer.** I build **local-first AI agents** — agents that actually *do the work*: break down tasks, call tools, stream every step live, remember what they learned, and turn repeat work into automation. All running on your machine, owned by you.

## 🤖 The Bodhi AI Ecosystem

An end-to-end, open-source desktop AI agent I design and build from the runtime up:

| Project | What it is |
|---|---|
| 🧘 **[Bodhi-AI](https://github.com/bigduu/Bodhi-AI)** | The desktop agent app — a Tauri shell over the Bamboo runtime + Lotus UI. Watch the agent think, act, and finish. |
| 🎋 **[Bamboo-agent](https://github.com/bigduu/Bamboo-agent)** | The heart: a local-first **AI agent runtime in Rust**. Persistent memory, 20+ built-in tools, skills, MCP, sub-agents, workflows & schedules behind one HTTP + WebSocket API. Embed it as a crate or run it as a server. |
| 🪷 **[Lotus](https://github.com/bigduu/Lotus)** / **[lotus-next](https://github.com/bigduu/lotus-next)** | The living interface for agents — React UI that streams reasoning, tool calls, and task lists live, turning a black-box AI into a transparent teammate you can watch and interrupt. `lotus-next` is the mobile-first shadcn rebuild. |
| ✨ **[Nova](https://github.com/bigduu/Nova)** | Give an LLM control of the **macOS desktop**: a single self-contained Rust binary speaking MCP — screenshots, Set-of-Mark targeting, Apple Vision OCR, mouse & keyboard. |
| 🏔 **[Zenith](https://github.com/bigduu/Zenith)** | The monorepo that ties it all together — runtime, UI, shell, Go backend, docs, and a coordinated release train. |
| 🏯 **[Pavilion](https://github.com/bigduu/Pavilion)** | Bodhi AI's front door — bilingual (中文/EN) website & docs with long-form architecture deep-dives. |

## 🧠 What I care about in agents

- **Local-first & private** — your model keys, your data, your machine; the cloud is optional
- **Transparency over magic** — every token, tool call, and decision streamed live to the UI
- **Memory that compounds** — persistent, searchable agent memory (CJK-aware BM25 + consolidation), not goldfish sessions
- **Real work, not chat** — task decomposition, sub-agents, schedules, workflows, and desktop control via MCP
- **Rust where it counts** — one fast, embeddable runtime instead of a stack of glue
